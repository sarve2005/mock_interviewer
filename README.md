# AI Mock Interviewer

An advanced AI-powered application designed to simulate technical interviews. It allows users to upload their resumes, receive tailored interview questions based on their experience, and get real-time feedback on their answers using Generative AI.

## Features

- **Resume Analysis**: Parses uploaded resumes (PDF) to understand the candidate's profile.
- **Dynamic Question Generation**: Generates context-aware technical questions using Google's Gemini models.
- **Real-time Interaction**:
  - **Speech-to-Text (STT)**: Transcribes user audio answers (future/planned integration).
  - **Text-to-Speech (TTS)**: Reads out questions using ElevenLabs AI audio.
- **Feedback & Scoring**: Provides detailed feedback and scores for every answer.
- **Authentication**: Secure user login and signup using Firebase Authentication.
- **History**: Saves interview sessions and progress (Firestore).

## Tech Stack

### Frontend
- **Framework**: React (Vite)
- **Styling**: Tailwind CSS
- **Routing**: React Router DOM
- **HTTP Client**: Axios
- **Icons**: Lucide React
- **Auth & DB**: Firebase SDK

### Backend
- **Framework**: FastAPI
- **Server**: Uvicorn
- **LLM**: Google Gemini (google-genai)
- **Vector Search**: FAISS & Sentence Transformers (for context retrieval)
- **PDF Processing**: PyPDF
- **TTS**: ElevenLabs
- **Database**: Firebase Admin SDK (Firestore)

## Prerequisites

- **Python** (3.9+)
- **Node.js** (16+)
- **Google Gemini API Key**
- **ElevenLabs API Key** (optional, for TTS)
- **Firebase Project**: Service account credentials (`serviceAccountKey.json`) and web config.

## Setup Instructions

### 1. Clone the Repository

```bash
git clone <repository_url>
cd new_AI
```

### 2. Backend Setup

Navigate to the backend directory:

```bash
cd new_AI_backend
```

Create a virtual environment and activate it:

```bash
# Windows
python -m venv venv
.\venv\Scripts\activate

# Mac/Linux
python3 -m venv venv
source venv/bin/activate
```

Install dependencies:

```bash
pip install -r requirements.txt
```

**Configuration**:
1. Create a `.env` file in `new_AI_backend/` based on references (not provided in repo, manually add keys).
   - `GEMINI_API_KEY=...`
   - `ELEVENLABS_API_KEY=...`
2. Place your Firebase `serviceAccountKey.json` in `new_AI_backend/backend/` or valid path referenced in `firebase_config.py`.

Run the server:

```bash
# From new_AI_backend directory
cd backend
python -m uvicorn app:app --reload
```
The backend runs on `http://localhost:8000`.

### 3. Frontend Setup

Navigate to the frontend directory:

```bash
cd ../new_AI_frontend/react-app
```

Install dependencies:

```bash
npm install
```

**Configuration**:
1. Create a `.env` file in `react-app/`.
2. Add your Firebase web configuration keys:
   ```env
   VITE_FIREBASE_API_KEY=...
   VITE_FIREBASE_AUTH_DOMAIN=...
   VITE_FIREBASE_PROJECT_ID=...
   VITE_FIREBASE_STORAGE_BUCKET=...
   VITE_FIREBASE_MESSAGING_SENDER_ID=...
   VITE_FIREBASE_APP_ID=...
   ```

Run the development server:

```bash
npm run dev
```
The frontend typically runs on `http://localhost:5173`.

## Usage

1. Open the frontend in your browser.
2. Sign up or Log in.
3. **Setup**: Upload your resume and configure interview settings (Role, Experience, Tech Stack).
4. **Interview**: Answer the questions generated by the AI. You can toggle audio availability.
5. **Feedback**: View detailed feedback and scoring after the interview.

## Project Structure

- `new_AI_backend/`: Python FastAPI application.
- `new_AI_frontend/`: React application.
- `uploads/`: Temporary storage for uploaded resumes (backend).
